🗄️ DB: Supabase PostgreSQL

Konsep & Arsitektur
🏗️
Stack Teknologi
Backend: Node.js + Express + TypeScript, di-deploy di Railway (Docker).
Database & Auth: Supabase PostgreSQL + Supabase Auth (JWT).
Push Notifikasi: OneSignal — dikirim ke perangkat nasabah saat antrian dipanggil.
WhatsApp: Baileys (WA Web multi-device) — kirim notifikasi WA ke nasabah.
Mobile: Android Studio memanggil endpoint /api/mobile/*.
🗂️
Multi-Cabang
Setiap cabang punya antrian independen. Staff hanya melihat antrian di cabang mereka. Nasabah memilih cabang saat daftar. Admin bisa melihat semua cabang sekaligus.

Contoh: CBG1 — Cabang Sudirman, CBG2 — Cabang Gunung Ibul.
🎫
Alur Antrian
1. Nasabah ambil antrian via mobile → status menunggu.
2. Staff panggil berikutnya → status → dilayani.
3. Push notif (OneSignal) + WA otomatis dikirim ke nasabah.
4. Staff tandai selesai atau batal.
5. Nomor antrian di-reset ke 1 setiap hari (midnight).

Autentikasi
🪙
Bearer Token (JWT dari Supabase)
Setelah login, simpan token dari response. Kirim di setiap request sebagai header:

Authorization: Bearer <token>

Token berlaku 1 jam. Setelah expired, login ulang.
Publik Tidak butuh token   Staff Token Teller atau CS   Teller Token Teller saja   Nasabah Token dari mobile login   Admin Token akun Admin 
// Contoh request dengan token
fetch('https://antrianbank.site/api/antrian/list', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
})

Auth
Endpoint registrasi, login, dan info user aktif.
POST /api/auth/register Publik ▼
Daftarkan akun Teller atau CS. Email harus berakhiran @teller.com atau @cs.com.
Request Body
{
  "email": "john@teller.com",
  "password": "password123",
  "nama": "John Doe",
  "cabang_id": 1,
  "no_loket": "T01"
}
Response 201
{
  "success": true,
  "data": {
    "token": "eyJ...",
    "user": {
      "id": "uuid",
      "role": "teller"
    }
  }
}
POST /api/auth/login Publik ▼
Login Teller, CS, dan Admin. Mengembalikan JWT token.
Request Body
{
  "email": "john@teller.com",
  "password": "password123"
}
Response 200
{
  "success": true,
  "data": {
    "token": "eyJhbGci...",
    "user": {
      "nama": "John Doe",
      "role": "teller",
      "no_loket": "T01",
      "cabang_id": 1

Semua endpoint mengembalikan data cabang sesuai staff yang login. Teller hanya lihat antrian Teller, CS hanya lihat antrian CS.

409ConflictData sudah ada (misal: email terdaftar)
