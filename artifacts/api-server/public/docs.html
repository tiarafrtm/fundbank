📋 REST API

Dashboard Teller API

Dokumentasi lengkap REST API sistem antrian FUND BANK. Digunakan oleh Dashboard Admin/Teller/CS dan aplikasi mobile nasabah (Android Studio).

🌐 Base URL: https://antrianbank.site
📦 Format: JSON
🔐 Auth: Bearer JWT
🗄️ DB: Supabase PostgreSQL

Konsep & Arsitektur
Bagaimana sistem ini dirancang dan bekerja dari ujung ke ujung.
🏗️
Stack Teknologi
Backend: Node.js + Express + TypeScript, di-deploy di Railway (Docker).
Database & Auth: Supabase PostgreSQL + Supabase Auth (JWT).
Push Notifikasi: OneSignal — dikirim ke perangkat nasabah saat antrian dipanggil.
WhatsApp: Baileys (WA Web multi-device) — kirim notifikasi WA ke nasabah.
Mobile: Android Studio (Java/Kotlin) memanggil endpoint /api/mobile/*.
🗂️
Multi-Cabang (Multi-Branch)
Setiap cabang (cabang) punya antrian independen. Staff hanya melihat & melayani antrian di cabang mereka sendiri. Nasabah memilih cabang saat daftar di mobile app. Admin bisa melihat data semua cabang sekaligus.

Contoh cabang: CBG1 — Cabang Sudirman, CBG2 — Cabang Gunung Ibul.
🎫
Alur Antrian
1. Nasabah ambil antrian via mobile (POST /api/mobile/antrian/ambil) → status menunggu.
2. Staff panggil antrian berikutnya (PUT /api/antrian/panggil) → status berubah ke dilayani.
3. Push notif (OneSignal) + WA otomatis dikirim ke nasabah.
4. Staff menandai selesai (PUT /api/antrian/selesai/:id) atau batal.
5. Nomor antrian di-reset ke 1 setiap hari (midnight).

Autentikasi
Cara menyertakan token di setiap request yang membutuhkan autentikasi.
🪙
Bearer Token (JWT dari Supabase)
Setelah login, simpan token dari response. Kirimkan di setiap request sebagai HTTP header:

Authorization: Bearer <token>

Token berlaku selama 1 jam. Setelah expired, login ulang untuk mendapatkan token baru.
Arti label auth di setiap endpoint:
Publik — Tidak butuh token
Staff — Token Teller atau CS
Teller — Token Teller saja
Nasabah — Token dari login mobile
Admin — Token akun Admin 
// Contoh request dengan autentikasi
fetch('https://antrianbank.site/api/antrian/list', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
})

Auth

POST /api/auth/register Publik Daftarkan akun Staff ▼
Mendaftarkan akun Teller atau CS baru. Email harus berakhiran @teller.com atau @cs.com.
Request Body
{
  "email": "john@teller.com",
  "password": "password123",
  "nama": "John Doe",
  "cabang_id": 1,
  "no_loket": "T01"
}
Response 201
{
  "success": true,
  "message": "Akun berhasil dibuat",
  "data": {
    "token": "eyJ...",
    "user": {
      "id": "uuid",
      "email": "john@teller.com",
      "nama": "John Doe",
      "role": "teller"
    }
  }
}

Format Response
Semua response dari API menggunakan format JSON yang konsisten.
Response Sukses
{
  "success": true,
  "message": "Pesan opsional",
  "data": { ... }
  // data bisa object, array, atau {}
}
Response Error
{
  "success": false,
  "message": "Pesan error detail",
  "data": {}
}
⚠️
Kode Error HTTP
HTTP status code yang digunakan oleh API ini.
KodeNamaArti
200OKRequest berhasil
201CreatedData berhasil dibuat (POST)
400Bad RequestData request tidak lengkap atau tidak valid
401UnauthorizedToken tidak ada, expired, atau tidak valid
403ForbiddenRole tidak punya izin (misal: nasabah akses endpoint admin)
404Not FoundData yang diminta tidak ditemukan
409ConflictData sudah ada (misal: email sudah terdaftar)
500Internal Server ErrorError di sisi server, hubungi developer
